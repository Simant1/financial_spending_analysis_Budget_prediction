What to say in the meeting

  1. Project introduction: This system implements my dissertation proposal for financial spending analysis and budget prediction using uploaded CSV or Excel records.
  2. Problem statement: Existing personal finance tools often hide their analysis process, but this system makes cleaning, transformation, anomaly detection, and forecasting visible and explainable.
  3. Pipeline explanation: The data first goes through quality assessment, then cleaning, transformation, descriptive analytics, anomaly detection, and finally ARIMA forecasting.
  4. Research value: The system combines practical software engineering with analytical methods, so it is both a usable application and an evidence-based dissertation artefact.
  5. Technical justification: Django is used for the secure web platform, Pandas and NumPy for preprocessing, statistical z-score logic for anomaly detection, and ARIMA for time-series prediction.
  6. Demonstration plan: I will upload a sample dataset, show the cleaning and quality pipeline, demonstrate dashboard filters, open the forecast comparison chart, and export the summary report.
  7. Future work: The next stage is to add smarter model selection, user-specific budgeting recommendations, and more advanced forecasting benchmarks.
